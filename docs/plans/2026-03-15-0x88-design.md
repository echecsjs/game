# 0x88 Board Representation Design

**Date:** 2026-03-15 **Status:** Approved

---

## Problem

The current flat `[64]` board representation has two performance bottlenecks in
`src/moves.ts`:

1. **Off-board check** — `collectSlidingAttacks` uses 4 comparisons per step:
   `rank >= 1 && rank <= 8 && file >= 1 && file <= 8`

2. **Attack detection** — `isSquareAttackedBy` iterates all 64 squares, calls
   `generateAttackMovesForSquare` for each attacker (building a full target
   list), then scans that list with `includes()`. O(64 × ray length × list scan)
   per check.

These compound badly: `isInCheck` is called for every pseudo-legal move during
legality filtering, so the cost multiplies across the whole move generation
pipeline.

Research confirms that **0x88 outperforms BigInt bitboards in JavaScript by
~40%** (TalkChess, Nov 2025). GarboChess-JS (0x88) achieves ~100k NPS vs
TomitankChess (BigInt bitboards) at 40–50k NPS — a mailbox engine wins in JS
because `BigInt` arithmetic is slow compared to native `Number` operations.

---

## Decision

Adopt the **0x88 board representation** throughout `FenState.board`. Add
precomputed `ATTACKS` and `RAYS` tables to `src/moves.ts`. Replace the 4-
comparison bounds check with `index & 0x88`. Replace ray-scanning attack
detection with a single table lookup per enemy piece.

---

## 0x88 Layout

```
index = rank * 16 + file
  where rank: 0-based top-down (rank 8 = 0, rank 1 = 7)
  where file: a=0 … h=7

a8=0,   b8=1,   … h8=7
a7=16,  b7=17,  … h7=23
…
a1=112, b1=113, … h1=119
```

Valid squares occupy indices `0..119` where `index & 0x88 === 0`. Invalid
(padding) slots have `index & 0x88 !== 0`. Array size: `128`.

**Off-board check:** `index & 0x88` — one bitwise AND.

- Indices ≥ 128 set the high bit → caught
- Indices < 0 (negative) set the sign bit in 32-bit representation → caught
- Wrap-around across file boundaries sets the 4th bit → caught

## ATTACKS and RAYS Tables

### Piece type bitmasks

```typescript
const PAWN_MASK = 0x01;
const KNIGHT_MASK = 0x02;
const BISHOP_MASK = 0x04; // also queen diagonals
const ROOK_MASK = 0x08; // also queen straights
const KING_MASK = 0x10;
```

Queen has no separate mask — inherits `BISHOP_MASK | ROOK_MASK`.

### Table size and index

Differences between any two valid 0x88 squares range from −119 to +119. Adding
119 shifts all values to `0..238`. Array size: 240 (one extra for safety,
matching chess.js).

```typescript
tableIndex = to - from + 119;
```

### Initialization (runs once at module load)

```typescript
const ATTACKS = new Uint8Array(240);
const RAYS = new Int8Array(240);

// Knight — 8 fixed offsets
for (const offset of [-33, -31, -18, -14, 14, 18, 31, 33]) {
  ATTACKS[offset + 119] |= KNIGHT_MASK;
}

// King — 8 adjacent offsets
for (const offset of [-17, -16, -15, -1, 1, 15, 16, 17]) {
  ATTACKS[offset + 119] |= KING_MASK;
}

// Pawns — white attacks +15/+17, black attacks -15/-17 (shared bit)
for (const offset of [15, 17]) {
  ATTACKS[offset + 119] |= PAWN_MASK;
  ATTACKS[-offset + 119] |= PAWN_MASK;
}

// Sliding pieces — walk every ray from every valid square
for (let from = 0; from <= 119; from++) {
  if (from & 0x88) continue;

  for (const dir of [-16, 16, -1, 1]) {
    // rook directions
    let to = from + dir;
    while (!(to & 0x88)) {
      const diff = to - from;
      ATTACKS[diff + 119] |= ROOK_MASK;
      RAYS[diff + 119] = dir;
      to += dir;
    }
  }

  for (const dir of [-17, -15, 15, 17]) {
    // bishop directions
    let to = from + dir;
    while (!(to & 0x88)) {
      const diff = to - from;
      ATTACKS[diff + 119] |= BISHOP_MASK;
      RAYS[diff + 119] = dir;
      to += dir;
    }
  }
}
```

### `isSquareAttackedBy` using the tables

```typescript
function isSquareAttackedBy(board, targetIndex, attackerColor): boolean {
  for (let i = 0; i <= 119; i++) {
    if (i & 0x88) continue;
    const piece = board[i];
    if (!piece || piece.color !== attackerColor) continue;

    const diff = i - targetIndex;
    const index = diff + 119;
    const mask = ATTACKS[index];
    if (!mask) continue;

    const pieceMask = PIECE_MASKS[piece.type];
    if (!(mask & pieceMask)) continue;

    // Pawn: direction must match attacker color
    if (piece.type === 'p') {
      if (attackerColor === 'w' && diff > 0) continue;
      if (attackerColor === 'b' && diff < 0) continue;
      return true;
    }

    // Knight / King: no blockers
    if (piece.type === 'n' || piece.type === 'k') return true;

    // Sliding piece: check for blockers
    const step = RAYS[index];
    let j = i + step;
    while (j !== targetIndex) {
      if (board[j] !== undefined) break;
      j += step;
    }
    if (j === targetIndex) return true;
  }
  return false;
}
```

## Module Changes

### `src/board.ts`

- `squareToIndex` / `indexToSquare` rewritten for 0x88 layout
- `INITIAL_BOARD` becomes `(Piece | undefined)[128]` with pieces at correct 0x88
  indices
- Export `OFF_BOARD = 0x88` constant for use in `moves.ts`
- `cloneBoard` unchanged

### `src/moves.ts`

- Add `ATTACKS`, `RAYS`, `PIECE_MASKS` constants (initialized at module load)
- Rewrite `isSquareAttackedBy` using the lookup tables
- Replace 4-comparison bounds check in `collectSlidingAttacks` with
  `index & 0x88`
- Update all index arithmetic throughout to use 0x88 layout
- Function signatures unchanged: `generateMoves`, `isInCheck`,
  `applyMoveToState`

### `src/fen.ts`

- `parseFen` board loop: iterate by rank/file, compute `rank * 16 + file`
- `serialiseFen` board loop: same pattern, skip padding slots
- Everything else unchanged

### Unchanged

- `src/types.ts`, `src/detection.ts`, `src/game.ts`, `src/index.ts`
- Public API — no visible changes to callers

## Expected Benchmark Impact

The `Game` cache means repeated queries are already O(1). The 0x88 gain shows in
cache population (first call per position) and `move()+undo()`:

| Operation            | Expected improvement                                    |
| -------------------- | ------------------------------------------------------- |
| `move()+undo()`      | Faster `applyMoveToState` via `& 0x88` + ATTACKS lookup |
| First `moves()` call | Faster `generateMoves` — bounds + attack detection      |
| Raw perft (no cache) | Most visible — exercises both hot paths without caching |

A raw perft benchmark using `applyMoveToState` directly (bypassing `Game`) will
be added to `comparison.bench.ts` to make the improvement measurable.
